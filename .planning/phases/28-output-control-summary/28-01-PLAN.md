<feature>
  <name>printSummary — compact human-readable index summary</name>
  <files>src/cli/summary.ts, tests/cli/summary.test.ts</files>
  <behavior>
    SUM-01: Header line format (includes TOTAL repos count)
    - Given 380 success, 15 skipped, 5 error results (total = 400) and 42300ms elapsed
    - printSummary writes: "Indexing complete: 400 repos (380 indexed, 15 skipped, 5 errors) in 42.3s\n"
    - Given 1 success, 0 skipped, 0 errors (total = 1) and 1500ms elapsed
    - printSummary writes: "Indexing complete: 1 repo (1 indexed, 0 skipped, 0 errors) in 1.5s\n"
    - "repo" is singular when total is 1, "repos" otherwise
    - Total = results.length (sum of all statuses). This is the explicit "total repos" field required by SUM-01.

    SUM-02: Error listing
    - Given an ErrorCollector with errors, printSummary calls errors.printSummary(stream) AFTER the header line
    - Given an ErrorCollector with no errors, printSummary does NOT call printSummary on it (no empty "Errors:" section)
    - The error detail is written to the SAME stream as the header (so everything appears together)

    SUM-03: Compactness
    - Given 400 results (380 success, 15 skipped, 5 errors), the header is exactly 1 line
    - The function does NOT print per-repo success/skip lines (those go in --verbose mode which is Plan 02's scope)
    - Total output for a 400-repo run with 5 errors: header line + ErrorCollector's grouped output (typically 10-15 lines max)
  </behavior>
  <implementation>
    Create `src/cli/summary.ts` exporting:

    ```typescript
    export function printSummary(
      stream: NodeJS.WriteStream,
      results: IndexResult[],
      elapsedMs: number,
      errors: ErrorCollector,
    ): void
    ```

    Implementation:
    1. Compute total = results.length.
    2. Count indexed/skipped/error from `results.filter(r => r.status === ...)`.
    3. Format elapsed as `(elapsedMs / 1000).toFixed(1)`.
    4. Write header line: `Indexing complete: {total} repo(s) ({indexed} indexed, {skipped} skipped, {errors} errors) in {T}s\n`.
    5. If `errors.hasErrors()`, write a blank line separator then call `errors.printSummary(stream)`.
    6. No color, no tables, no per-repo detail. Just the counts and error list.

    Import IndexResult from `../../indexer/pipeline.js` and ErrorCollector from `../../indexer/progress.js`.
  </implementation>
</feature>
