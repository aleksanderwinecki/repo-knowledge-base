<task type="auto">
  <name>Task 1: Repo scanner, git helper, and metadata extractor</name>
  <files>src/indexer/scanner.ts, src/indexer/git.ts, src/indexer/metadata.ts</files>
  <action>
Create three extractor modules. All are pure functions -- no database access.

1. Create `src/indexer/scanner.ts`:
   - `discoverRepos(rootDir: string): string[]` -- scans immediate children of rootDir (NOT recursive).
   - A directory is a repo if: `fs.existsSync(path.join(dir, '.git'))` AND at least one of these exists: `mix.exs`, `package.json`, `Gemfile`, `Cargo.toml`, `go.mod`.
   - Skip directories named: `node_modules`, `.git`, `_build`, `deps`, `vendor`, `dist`.
   - Return sorted array of absolute paths.
   - Use `fs.readdirSync(rootDir, { withFileTypes: true })` and check `entry.isDirectory()`.
   - If rootDir doesn't exist, throw a clear error.

2. Create `src/indexer/git.ts`:
   - `getCurrentCommit(repoPath: string): string | null` -- runs `git rev-parse HEAD` via `execSync({ cwd: repoPath, encoding: 'utf-8' })`. Returns trimmed SHA or null if git fails.
   - `getChangedFiles(repoPath: string, sinceCommit: string): { added: string[]; modified: string[]; deleted: string[] }` -- runs `git diff --name-status sinceCommit..HEAD`. Parses tab-separated lines where first column is A/M/D. Returns categorized file lists.
   - `isCommitReachable(repoPath: string, commitSha: string): boolean` -- runs `git cat-file -t commitSha`. Returns true if exit code 0 (commit exists), false otherwise.
   - All git functions wrap execSync in try/catch -- git failures return null/false/empty, never throw.

3. Create `src/indexer/metadata.ts`:
   - Define `RepoMetadata` interface: `{ name: string; path: string; description: string | null; techStack: string[]; keyFiles: string[]; currentCommit: string | null }`.
   - `extractMetadata(repoPath: string): RepoMetadata` -- orchestrates all metadata extraction for a single repo.
   - `name`: `path.basename(repoPath)`.
   - `description`: Read `README.md` first 500 chars, extract first paragraph after title (skip `# Title` line, take text until next `##` or blank line). If no README, try `CLAUDE.md` same approach. If neither, null.
   - `techStack`: Check file existence: `mix.exs` -> 'elixir', `package.json` -> 'node', `Gemfile` -> 'ruby', `Cargo.toml` -> 'rust', `go.mod` -> 'go'. Then parse dep files for key deps:
     - mix.exs: regex `{:(\w+),` to find dep atoms. Match against known list: phoenix, ecto, absinthe, broadway, oban, commanded, eventstore, grpc.
     - package.json: JSON.parse, merge `dependencies` + `devDependencies` keys. Match against: express, fastify, next, react, vue, angular, nestjs.
     - Wrap each dep file parse in try/catch (corrupted files shouldn't crash).
   - `keyFiles`: Hardcoded check for: README.md, CLAUDE.md, AGENTS.md, mix.exs, package.json, Gemfile, config/ (as dir). Plus list top-level directories: lib/, priv/, test/, tests/, proto/, src/, apps/. Only include files/dirs that actually exist.
   - `currentCommit`: Call `getCurrentCommit(repoPath)` from git.ts.
  </action>
  <verify>
    <automated>npx tsc --noEmit && echo "TypeScript compiles"</automated>
  </verify>
  <done>scanner.ts exports discoverRepos that finds repos with .git + project file. git.ts exports getCurrentCommit, getChangedFiles, isCommitReachable with safe error handling. metadata.ts exports extractMetadata returning name, description, techStack, keyFiles, currentCommit.</done>

<task type="auto">
  <name>Task 2: Database writer with transactional persistence and FTS sync</name>
  <files>src/indexer/writer.ts, src/index.ts</files>
  <action>
Create the database writer module that persists extracted data in transactions, then update exports.

1. Create `src/indexer/writer.ts`:
   - Import Database type from better-sqlite3, import indexEntity/removeEntity from `../db/fts.js`.
   - Define `RepoData` interface: `{ metadata: RepoMetadata; modules?: ElixirModule[]; events?: ProtoDefinition[]; edges?: EdgeData[] }` -- modules/events/edges are optional (Plan 02 fills these in).
   - `upsertRepo(db, metadata: RepoMetadata): number` -- INSERT OR REPLACE into repos table (name, path, description, last_indexed_commit). Uses `db.prepare("INSERT INTO repos (...) VALUES (...) ON CONFLICT(name) DO UPDATE SET ...").run(...)`. Returns the repo id (use `lastInsertRowid` or query by name).
   - `clearRepoEntities(db, repoId: number): void` -- Deletes all files, modules, events, and edges belonging to this repo. Also removes corresponding FTS entries. Order: edges first (no FK), then events, modules, files (CASCADE handles some, but be explicit for FTS cleanup).
   - `persistRepoData(db, data: RepoData): { repoId: number }` -- Transaction-wrapped. Calls upsertRepo, clearRepoEntities, then inserts new data: for each module -> insert into modules table + indexEntity for FTS, for each event -> insert into events + indexEntity, for each edge -> insert into edges. Updates last_indexed_commit. Also indexes the repo itself in FTS (repo name + description).
   - `clearRepoFiles(db, repoId: number, filePaths: string[]): void` -- For incremental indexing: removes entities extracted from specific files (not full repo clear). Queries modules/events by file reference, removes their FTS entries, then deletes the records.
   - All database operations use prepared statements (db.prepare) for performance.

2. Update `src/index.ts`:
   - Add exports for new modules: `discoverRepos` from scanner, `extractMetadata` and `RepoMetadata` from metadata, `getCurrentCommit`/`getChangedFiles`/`isCommitReachable` from git, `persistRepoData`/`clearRepoEntities` from writer.
  </action>
