  <name>Task 1: Embedding pipeline and batch generation orchestrator</name>
  <files>src/embeddings/pipeline.ts, src/embeddings/generate.ts, tests/embeddings/pipeline.test.ts</files>
  <behavior>
    - getEmbeddingPipeline() returns a FeatureExtractionPipeline (lazy-loaded, cached singleton)
    - generateEmbedding("search_document: test text") returns a Float32Array of length 256
    - generateEmbeddingsBatch(["text1", "text2"]) returns Float32Array[] of length 2, each with 256 dims
    - Batch embedding applies "search_document: " prefix, mean pooling, layer_norm, slice to 256d, L2 normalize
    - generateAllEmbeddings(db, false) embeds only entities missing from entity_embeddings table
    - generateAllEmbeddings(db, true) embeds all entities (force mode)
    - generateAllEmbeddings returns count of embeddings generated
    - generateAllEmbeddings returns 0 when isVecAvailable() is false
    - Individual entity embedding failure is logged and skipped (does not crash batch)
    - Entities are processed in chunks of 32
  </behavior>
  <action>
    Create `src/embeddings/pipeline.ts`:
    - Import `pipeline`, `layer_norm`, `env` from `'@huggingface/transformers'`
    - Set `env.cacheDir` to `path.join(os.homedir(), '.kb', 'models')` for stable caching across reinstalls (per research open question)
    - Implement singleton: `let extractor: FeatureExtractionPipeline | null = null`
    - `getEmbeddingPipeline()`: lazy-create with `pipeline('feature-extraction', 'nomic-ai/nomic-embed-text-v1.5', { dtype: 'fp32' })`, cache in `extractor`
    - `generateEmbedding(text: string): Promise<Float32Array>`: prefix with `"search_document: "`, run pipeline with `{ pooling: 'mean' }`, apply `layer_norm(output, [output.dims[1]])`, `.slice(null, [0, 256])`, `.normalize(2, -1)`, return `new Float32Array(output.data)` (slice first 256 elements from the typed array data)
    - `generateEmbeddingsBatch(texts: string[]): Promise<Float32Array[]>`: prefix each text with `"search_document: "`, run pipeline in batch, apply same Matryoshka truncation, return array of Float32Array (one per text, each 256 elements). Use `output.data.slice(j * 256, (j + 1) * 256)` for row extraction from the batched tensor.
    - CONST `MATRYOSHKA_DIM = 256` at module level

    Create `src/embeddings/generate.ts`:
    - Import `isVecAvailable` from `'../db/vec.js'`
    - Import `composeEmbeddingText` from `'./text.js'`
    - Import `generateEmbeddingsBatch` from `'./pipeline.js'`
    - Import type `Database` from `'better-sqlite3'`
    - CONST `CHUNK_SIZE = 32`

    `generateAllEmbeddings(db: Database.Database, force: boolean): Promise<number>`:
    - If `!isVecAvailable()` return 0
    - Query all entities to embed from DB (modules, events, services, repos, learned_facts)
    - For each entity, create an EmbeddingEntity object from DB row columns
    - If `!force`, filter out entities that already exist in entity_embeddings (query: `SELECT entity_type, entity_id FROM entity_embeddings`)
    - Call `composeEmbeddingText` for each entity, skip if null
    - Process in chunks of CHUNK_SIZE:
      - Call `generateEmbeddingsBatch(texts)` for the chunk
      - Use a sync `db.transaction()` to batch-INSERT all embeddings into entity_embeddings (Pitfall 4: async generate first, then sync persist)
      - INSERT: `INSERT INTO entity_embeddings(embedding, entity_type, entity_id) VALUES (?, ?, ?)` with Float32Array binding
    - Wrap each chunk in try/catch -- on error, log warning with chunk index and continue to next chunk
    - Return total count of embeddings inserted
    - Log "Downloading embedding model (first run only)..." before first pipeline init (Pitfall 5)

    Create `tests/embeddings/pipeline.test.ts`:
    - Test `generateEmbedding` returns Float32Array of length 256 (requires model download -- use a short text)
    - Test `generateEmbeddingsBatch` returns correct count and dimensions
    - Test output is normalized (L2 norm approximately 1.0)
    - Test `generateAllEmbeddings` returns 0 when vec is unavailable (mock `isVecAvailable`)
    - NOTE: These tests require model download on first run (~100MB). They are integration-level tests. Mark with a describe block that can be filtered if needed. Consider a `SKIP_EMBEDDING_MODEL` env var check for CI environments without network.

    Install dependency: `npm install @huggingface/transformers`

    ```
    // === Phase 4: Embedding generation (async, after all persistence) ===
    if (success > 0 && isVecAvailable()) {
      try {
        const startMs = Date.now();
        const embeddingCount = await generateAllEmbeddings(db, options.force);
        const elapsed = Date.now() - startMs;
        console.log(`Generated ${embeddingCount} embeddings (${elapsed}ms)`);
      } catch (error) {
        const msg = error instanceof Error ? error.message : String(error);
        console.warn(`Embedding generation failed: ${msg}`);
      }

      console.log('sqlite-vec not available, skipping embeddings');
    }
    ```
