    - After V8 migration with sqlite-vec available: entity_embeddings table exists as vec0 virtual table with float[256] embedding column, entity_type text column, entity_id integer column
    - After V8 migration without sqlite-vec: entity_embeddings table does NOT exist, no error
    - SCHEMA_VERSION is 8
    - Migrating from V7 to V8 preserves all existing data
    - clearRepoEmbeddings(db, repoId) removes embeddings for all entity types belonging to that repo (modules, events, services, repo itself)
    - clearRepoEmbeddings is a no-op when sqlite-vec is unavailable

  <action>
    Create `src/db/vec.ts` with:
    - `loadVecExtension(db: Database.Database): boolean` -- calls `sqliteVec.load(db)` in try/catch, sets module-level `vecAvailable` flag, returns success boolean
    - `isVecAvailable(): boolean` -- returns the flag
    - Import `sqlite-vec` as `* as sqliteVec` (the npm package uses `load(db)` API per research)

    Modify `src/db/database.ts`:
    - Import `loadVecExtension` from `./vec.js`
    - Call `loadVecExtension(db)` AFTER pragmas but BEFORE `initializeSchema(db)` -- critical ordering per Pitfall 1

    - Add V8 step to `runMigrations`: `if (fromVersion < 8 && toVersion >= 8) { migrateToV8(db); }`

    Modify `src/db/schema.ts`:
    - Change `SCHEMA_VERSION` from 7 to 8

    Modify `src/indexer/writer.ts`:
    - Import `isVecAvailable` from `../db/vec.js`
    - Add `clearRepoEmbeddings(db: Database.Database, repoId: number): void` -- if `!isVecAvailable()` return; otherwise for each entity type (module, event, service, repo): query entity IDs belonging to repoId, then DELETE from entity_embeddings WHERE entity_type = ? AND entity_id = ?
    - Call `clearRepoEmbeddings` inside `clearRepoEntities` before deleting the entities themselves (embeddings reference entity IDs, so clear embeddings first)

    Create `tests/db/vec.test.ts`:
    - Test loadVecExtension + isVecAvailable returns true (assumes sqlite-vec is installed in dev deps)
    - Test vec0 table creation after migration
    - Test entity_embeddings accepts Float32Array insertion and KNN query
    - Test clearRepoEmbeddings removes the right rows

    Update `tests/db/schema.test.ts`:

    - composeEmbeddingText({entityType: 'repo', name: 'app-bookings', description: 'Booking management service'}) returns tokenized repo text
    - composeEmbeddingText({entityType: 'learned_fact', content: 'Bookings use soft deletes', repo: 'app-bookings'}) returns tokenized fact text
