<task type="auto" tdd="true">
  <name>Task 2: searchHybrid with RRF scoring, withAutoSyncAsync, and search index exports</name>
  <files>src/search/hybrid.ts, src/search/index.ts, src/mcp/sync.ts, tests/search/hybrid.test.ts</files>
  <behavior>
    - searchHybrid combines FTS5 results and KNN results using RRF with k=60
    - searchHybrid returns FTS5-only results when searchSemantic returns [] (graceful degradation)
    - searchHybrid deduplicates entities that appear in both FTS5 and KNN (key: entityType:entityId)
    - searchHybrid entities appearing in both lists get higher RRF score than single-list entities
    - searchHybrid uses 1-indexed ranks in RRF formula: 1/(k + i + 1) for loop index i
    - searchHybrid respects limit option
    - searchHybrid results are sorted by combined RRF score descending
    - withAutoSyncAsync properly awaits async queryFn before extracting repo names
  </behavior>
  <action>
1. **Create `src/search/hybrid.ts`:**
   - `searchHybrid(db, query, options?)` -> `Promise<TextSearchResult[]>`
   - Runs both `searchText(db, query, { ...options, limit: limit * 2 })` (sync) and `await searchSemantic(db, query, { ...options, limit: limit * 2 })` (async)
   - If vecResults is empty, return `ftsResults.slice(0, limit)` (graceful degradation -- this covers both vec-unavailable and empty-embeddings cases since searchSemantic handles those)
   - RRF merge: `const scores = new Map<string, { score: number; result: TextSearchResult }>()`
   - Key format: `${r.entityType}:${r.entityId}`
   - For each list, score entry `i` as `1 / (RRF_K + i + 1)` where `RRF_K = 60` (1-indexed rank)
   - If key already exists in map, accumulate score (handles dedup)
   - Sort by score descending, take top `limit` results
   - Set `relevance` to the RRF score on each result
   - Export `RRF_K` constant for test access

2. **Add `withAutoSyncAsync` to `src/mcp/sync.ts`:**
   - Same as `withAutoSync` but `queryFn: () => Promise<T>` and awaits the result before calling `extractRepoNames`
   - Signature: `async function withAutoSyncAsync<T>(db, queryFn: () => Promise<T>, extractRepoNames: (result: T) => string[]): Promise<T>`
   - `let result = await queryFn()` then extract repo names, sync, `if synced: result = await queryFn()`
   - Export it alongside existing `withAutoSync`
