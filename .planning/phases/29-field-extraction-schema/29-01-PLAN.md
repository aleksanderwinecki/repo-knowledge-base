1. In `src/indexer/graphql.ts`:

Add exported interface and function:
```typescript
export interface GraphqlField {
  name: string;
  type: string;  // Full type expression including !, [], [!], etc.
}

export function parseGraphqlFields(body: string): GraphqlField[] {
  const fields: GraphqlField[] = [];
  const fieldRe = /^\s*(\w+)(?:\([^)]*\))?\s*:\s*(\[?\w+!?\]?!?)/gm;
  let match;
  while ((match = fieldRe.exec(body)) !== null) {
    fields.push({ name: match[1]!, type: match[2]! });
  }
  return fields;
}
```

The regex matches:
- `fieldName: TypeName` (basic)
- `fieldName: TypeName!` (non-null)
- `fieldName: [TypeName]` (list)
- `fieldName: [TypeName!]!` (non-null list of non-null)
- `fieldName(arg: ArgType): ReturnType` (with arguments)

It naturally skips enum values (no `:` present) and comment lines (start with `#`, not `\w`).

Important: Only call `parseGraphqlFields` for types with `kind` in `['type', 'input', 'interface']`. Skip `enum`, `union`, `scalar` -- they don't have field declarations (see Pitfall 6 in RESEARCH.md).
