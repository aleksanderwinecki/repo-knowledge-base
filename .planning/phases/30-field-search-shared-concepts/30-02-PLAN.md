    **2. Add `'field'` case to `getEntitiesByExactName` in `src/search/entity.ts`:**
    ```typescript
    case 'field': {
      let sql = `SELECT f.id, f.field_name as name, f.parent_type, f.parent_name,
                        f.field_type, f.nullable, f.source_file,
                        r.name as repo_name, r.path as repo_path
                 FROM fields f JOIN repos r ON f.repo_id = r.id
                 WHERE f.field_name = ?`;
      const params: (string | number)[] = [name];
      if (repoFilter) { sql += ' AND r.name = ?'; params.push(repoFilter); }
      sql += ' ORDER BY r.name, f.parent_type, f.parent_name';
      const rows = db.prepare(sql).all(...params) as Array<{
        id: number; name: string; parent_type: string; parent_name: string;
        field_type: string; nullable: number; source_file: string | null;
        repo_name: string; repo_path: string;
      }>;
      return rows.map((r) => ({
        id: r.id,
        name: r.name,
        repoName: r.repo_name,
        repoPath: r.repo_path,
        filePath: r.source_file,
        description: `${r.parent_type} ${r.parent_name}.${r.name}: ${r.field_type} (${r.nullable ? 'nullable' : 'required'})`,
      }));
    }
    ```

    **3. Add shared concept enrichment to `findExact`:**
    After building cards for field type, enrich the description of field cards with shared concept info. After the existing entity loop in findExact, if the filter type is 'field' and cards.length > 0, compute the cross-repo count:
    ```typescript
    // After the for(type of types) loop, enrich field cards with shared concept metadata
    if (cards.length > 0 && filters?.type === 'field') {
      const repoCount = new Set(cards.map(c => c.repoName)).size;
      if (repoCount >= 2) {
        for (const card of cards) {
          card.description = `[shared across ${repoCount} repos] ${card.description}`;
        }
      }
    }
    ```
    This approach uses the already-fetched cards to count distinct repos -- no extra query needed. The shared concept indicator is prepended to the description so it's visible in both MCP and CLI output without changing the EntityCard interface.
