  <behavior>
    ## Types (add to src/search/types.ts)

    GraphEdge: { targetRepoId: number, mechanism: string, confidence: string | null, via: string | null, relationshipType: string }
    ServiceGraph: { forward: Map<number, GraphEdge[]>, reverse: Map<number, GraphEdge[]>, repoNames: Map<number, string>, repoIds: Map<string, number> }
    BfsNode: { repoId: number, repoName: string, depth: number }
    GraphHop: { fromRepoId: number, fromRepoName: string, toRepoId: number, toRepoName: string, mechanism: string, confidence: string | null, via: string | null }

    ## buildGraph (GRAPH-01 + GRAPH-04)

    - buildGraph(db) returns ServiceGraph with forward and reverse adjacency lists
    - Direct repo-to-repo edges (calls_grpc, calls_http, routes_to) appear in both forward and reverse maps
    - Event-mediated edges: repo A produces event E, repo B consumes event E -> forward[A] includes edge to B with mechanism='event', via='EventName'
    - Kafka edges: repo A produces topic T, repo B consumes topic T -> forward[A] includes edge to B with mechanism='kafka', via='TopicName'
    - Self-loops from event/kafka resolution are excluded (producer == consumer)
    - Duplicate event edges (same from/to/event) are deduplicated
    - Duplicate kafka edges (same from/to/topic) are deduplicated
    - Multiple edges between same pair with different mechanisms are kept (e.g., gRPC + kafka)
    - Unresolved edges (target_type='service_name') included in forward map with targetRepoId=0
    - repoNames maps id->name, repoIds maps name->id for all repos
    - Confidence is extracted from metadata JSON

    ## bfsDownstream (GRAPH-02)

    - bfsDownstream(graph, repoId, maxDepth?) returns BfsNode[] of all reachable nodes via reverse adjacency (services that depend on this one)
    - Each node has correct depth (1 = direct dependent)
    - Does not include the starting node in results
    - Respects maxDepth limit (default: no limit)
    - Does not traverse from unresolved nodes (repoId=0)
    - Handles cycles without infinite loops (visited set)
    - Returns empty array for unknown repoId

    ## shortestPath (GRAPH-03)

    - shortestPath(graph, fromRepoId, toRepoId) returns GraphHop[] or null
    - Treats graph as undirected: traverses both forward and reverse edges
    - Returns null when no path exists
    - Returns empty array when from === to
    - Returns single hop for directly connected services
    - Returns shortest multi-hop path (BFS guarantees shortest)
    - Each hop has fromRepoName, toRepoName, mechanism, confidence, via
    - Hops show actual edge direction (who calls whom), not traversal direction
  </behavior>
  <implementation>
    ## Implementation Order

    1. **Add types to src/search/types.ts**: GraphEdge, ServiceGraph, BfsNode, GraphHop interfaces.

    2. **Create src/search/graph.ts** with three exported functions:

    **buildGraph(db):**
    - Load repos: `SELECT id, name FROM repos` -> build repoNames and repoIds maps
    - Load direct edges: `SELECT source_id, target_id, relationship_type, metadata FROM edges WHERE source_type='repo' AND target_type='repo'`
    - Load event producers: `SELECT DISTINCT e.source_id as repo_id, e.target_id as event_id, ev.name as event_name FROM edges e JOIN events ev ON ev.id = e.target_id WHERE e.relationship_type = 'produces_event'`
    - Load event consumers: `SELECT DISTINCT e.source_id as repo_id, e.target_id as event_id FROM edges e WHERE e.relationship_type = 'consumes_event'`
    - Load kafka producers: `SELECT source_id as repo_id, json_extract(metadata, '$.topic') as topic, metadata FROM edges WHERE relationship_type = 'produces_kafka' AND json_extract(metadata, '$.topic') IS NOT NULL`
    - Load kafka consumers: `SELECT source_id as repo_id, json_extract(metadata, '$.topic') as topic, metadata FROM edges WHERE relationship_type = 'consumes_kafka' AND json_extract(metadata, '$.topic') IS NOT NULL`
    - Load unresolved edges: `SELECT source_id, relationship_type, metadata FROM edges WHERE source_type='repo' AND target_type='service_name'`
    - Resolve events in JS: Map<eventId, consumerRepoIds[]>, iterate producers, create edges. Deduplicate by (from, to, eventName). Skip self-loops.
    - Resolve kafka in JS: Map<topic, consumerRepoIds[]>, iterate producers, create edges. Deduplicate by (from, to, topic). Skip self-loops.
    - Build forward Map<repoId, GraphEdge[]> and reverse Map<repoId, GraphEdge[]> from all resolved edges.
    - Unresolved edges: add to forward map only with targetRepoId=0, via=targetName from metadata.
    - Import extractConfidence, extractMetadataField from edge-utils.ts for metadata parsing.

    **bfsDownstream(graph, startRepoId, maxDepth?):**
    - Use graph.reverse adjacency list (services that call/depend on startRepoId)
    - Queue-based BFS with [repoId, depth] tuples
    - visited Set<number> starting with startRepoId
    - Skip repoId=0 (unresolved) in queue processing
    - Collect BfsNode[] results with depth
    - Respect maxDepth (default: Infinity)

    **shortestPath(graph, fromRepoId, toRepoId):**
    - If from === to, return []
    - BFS with parent Map<number, { fromId, edge, edgeIsForward }> for path reconstruction
    - At each node, collect neighbors from BOTH forward and reverse adjacency lists
    - For forward edges: neighbor = edge.targetRepoId, edgeIsForward = true
    - For reverse edges: the "neighbor" is the source of that edge (need to know who the reverse edge came from). In reverse map, graph.reverse.get(current) gives edges where current is the target. The edge.targetRepoId in reverse is actually the SOURCE repo. So neighbor = edge.targetRepoId, edgeIsForward = false.
    - Wait -- the reverse map: reverse.get(X) returns GraphEdge[] where each edge's targetRepoId is a repo that CALLS X. So for undirected traversal from current, forward.get(current) gives repos that current calls, reverse.get(current) gives repos that call current. Both are neighbors.
    - When found, reconstruct path from parent pointers. Each hop should show actual edge direction: if edgeIsForward, from=current, to=neighbor. If not forward (reverse edge), from=neighbor, to=current (the actual call direction).
    - Return null if target not reached.

    3. **Update src/search/index.ts**: Add exports for buildGraph, bfsDownstream, shortestPath, and type exports for GraphEdge, ServiceGraph, BfsNode, GraphHop.
