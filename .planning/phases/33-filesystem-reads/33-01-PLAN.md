<objective>
Replace git child process file I/O with direct filesystem reads in all extractors.

Purpose: Eliminate thousands of `execSync('git show')` and `execSync('git ls-tree')` calls during indexing. Each call has ~2-5ms overhead from process spawning. With ~700 repos and hundreds of files per repo, this is the primary performance bottleneck.

Output: New `listWorkingTreeFiles` and `readWorkingTreeFile` functions in git.ts; all 9 extractor files refactored to use them with `branch` parameter removed from every extractor signature.
</objective>

Add two new exported functions to `src/indexer/git.ts`. Do NOT remove `listBranchFiles` or `readBranchFile` yet (plan 02 handles the full wiring and test updates; removing prematurely would break compilation).

1. `listWorkingTreeFiles(repoPath: string): string[]`
   - Recursively walk the filesystem under `repoPath`
   - Return relative file paths (relative to repoPath), using forward slashes
   - Skip directories: `.git`, `node_modules`, `_build`, `deps`, `vendor`, `dist`, `.elixir_ls` (same as scanner.ts SKIP_DIRS)
   - Only return files (not directories)
   - Return empty array if path doesn't exist or isn't readable
   - Use `fs.readdirSync` with `{ withFileTypes: true, recursive: true }` (Node 18.17+ supports recursive option). If recursive option isn't available in the project's Node target, implement manual recursion with a stack/queue.
   - Important: the returned paths must use `/` separators (not `\`), matching the format that `listBranchFiles` returned from `git ls-tree --name-only`

2. `readWorkingTreeFile(repoPath: string, filePath: string): string | null`
   - Read `path.join(repoPath, filePath)` via `fs.readFileSync(fullPath, 'utf-8')`
   - Return null if file doesn't exist or is unreadable (wrap in try/catch)
   - Cap at 500KB (same as readBranchFile's maxBuffer: `500 * 1024`). If file size exceeds this, return null. Check size with `fs.statSync` before reading.

Import `fs` and `path` at the top of git.ts (they may already be partially imported; add what's missing).

Do NOT modify any existing functions in this file. Do NOT remove the old `listBranchFiles`/`readBranchFile` exports.
