  <action>
Create `src/mcp/format.ts` exporting:

```typescript
interface McpResponse<T> {
  summary: string;
  data: T[];
  total: number;
  truncated: boolean;
}

function formatResponse<T>(
  items: T[],
  summaryFn: (shown: T[]) => string,
  maxItems?: number,  // default 10
): string
```

Implementation:
1. Slice items to `maxItems` (default 10)
2. Build `McpResponse` object with summary from `summaryFn(sliced)`, data=sliced, total=items.length, truncated=(items.length > sliced.length)
3. JSON.stringify the response
4. If result.length > 4000 chars, recursively call with `Math.floor(maxItems / 2)` (minimum 1)
5. If even 1 item exceeds 4000, truncate the single item's string fields to fit
6. Return the JSON string

Write tests first in `tests/mcp/format.test.ts` following the project's existing test pattern (vitest, describe/it blocks). Test all behaviors listed above.

    - flagStaleFacts returns learned facts older than N days (default 90) with their repo association
    - flagStaleFacts does NOT delete anything
  </behavior>
  <action>
**sync.ts:**

Create `src/mcp/sync.ts` exporting `checkAndSyncRepos(db, repoNames: string[]): SyncResult`.

1. For each repoName, query `repos` table for `path` and `last_indexed_commit`
2. Skip if repo not in DB or path doesn't exist on disk
3. Call `getCurrentCommit(path)` — skip if null
4. Compare to stored `last_indexed_commit` — skip if same
5. Call `indexSingleRepo(db, path, { force: false, rootDir: path.dirname(path) })` for stale repos
6. Cap at `MAX_SYNC_PER_QUERY = 3` repos re-indexed per call
7. Return `{ synced: string[], skipped: string[] }` where skipped = stale repos beyond the cap

For testing: mock `getCurrentCommit` and `indexSingleRepo` using vitest's `vi.mock()` to avoid needing real git repos. Set up a real in-memory DB with repo rows to test the query/comparison logic.

**hygiene.ts:**

Create `src/mcp/hygiene.ts` exporting:
- `detectDeletedRepos(db): string[]` — queries all repos, filters by `!fs.existsSync(path)`
- `pruneDeletedRepos(db, repoNames: string[]): void` — for each repo: look up repoId, call existing `clearRepoEntities(db, repoId)`, then `DELETE FROM repos WHERE id = ?`
- `flagStaleFacts(db, maxAgeDays?: number): LearnedFact[]` — queries `learned_facts WHERE created_at < datetime('now', '-N days')`. Default maxAgeDays=90. Returns the facts without deleting.

For testing: use real temp-dir SQLite databases (project pattern). Insert test repos with fake paths, test that detectDeletedRepos finds them. Test pruneDeletedRepos actually removes all related data. Test flagStaleFacts with old timestamps.
