**Step 1: Create `src/mcp/server.ts`**

Start with `#!/usr/bin/env node` shebang (CRITICAL for bin entry — see Pitfall 5 in RESEARCH.md).

```typescript
#!/usr/bin/env node

import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import os from 'os';
import path from 'path';
import { openDatabase, registerShutdownHandlers } from '../db/database.js';
// Import all tool registration functions
import { registerSearchTool } from './tools/search.js';
// ... etc for all 7

const dbPath = process.env.KB_DB_PATH ?? path.join(os.homedir(), '.kb', 'knowledge.db');
const db = openDatabase(dbPath);
registerShutdownHandlers(db);

const server = new McpServer({ name: 'kb', version: '1.0.0' });

// Register all 7 tools
registerSearchTool(server, db);
registerEntityTool(server, db);
registerDepsTool(server, db);
registerLearnTool(server, db);
registerForgetTool(server, db);
registerStatusTool(server, db);
registerCleanupTool(server, db);

async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('[kb-mcp] server running on stdio');
}

main().catch((error) => {
  console.error('[kb-mcp] Fatal error:', error);
  process.exit(1);
});
```

CRITICAL: NEVER use `console.log()` anywhere in MCP server code — only `console.error()`. `console.log` writes to stdout and corrupts the JSON-RPC message stream.

**Step 2: Create 7 tool files under `src/mcp/tools/`**

Each tool follows this pattern:
1. Export a `registerXxxTool(server: McpServer, db: Database.Database): void` function
2. Call `server.registerTool(name, { description, inputSchema }, handler)`
3. Handler: try/catch wrapping existing API call, format with `formatResponse`, return `{ content: [{ type: 'text', text }] }`
4. On error: return `{ content: [{ type: 'text', text: message }], isError: true }`

**Tool specifications:**

**kb_search** (search.ts):
- Input: `query: z.string()`, `limit: z.number().min(1).max(50).optional()`, `repo: z.string().optional()`
- Calls `searchText(db, query, { limit: limit ?? 10, repoFilter: repo })`
- After getting results, extract unique repo names and call `checkAndSyncRepos(db, repoNames)`
- If any repos were synced, re-run the search to get fresh results
- Format with `formatResponse(results, (items) => \`Found ${total} results for "${query}"\`)`

**kb_entity** (entity.ts):
- Input: `name: z.string()`, `type: z.string().optional()`, `repo: z.string().optional()`
- Calls `findEntity(db, name, { type, repo })`
- Extract repo names from results, call `checkAndSyncRepos`
- If synced, re-run findEntity
- Format with `formatResponse`

**kb_deps** (deps.ts):
- Input: `name: z.string()`, `direction: z.enum(['upstream', 'downstream']).optional()`, `depth: z.number().min(1).max(10).optional()`, `repo: z.string().optional()`
- Calls `queryDependencies(db, name, { direction, depth, repo })`
- If result is null, return `{ content: [{ type: 'text', text: JSON.stringify({ summary: 'No entity found...', data: null }) }] }`
- Extract result.entity.repoName + all dependency repoNames, call `checkAndSyncRepos`
- If synced, re-run query
- Format the single DependencyResult (not an array — wrap in array for formatResponse or handle as special case)

**kb_learn** (learn.ts):
- Input: `content: z.string().describe('The fact to remember')`, `repo: z.string().optional().describe('Associate with a specific repo')`
- Calls `learnFact(db, content, repo)`
- Return JSON with `{ summary: 'Learned: "..." (id: N)', data: fact }`
- No auto-sync needed (write operation)

**kb_forget** (forget.ts):
- Input: `id: z.number().describe('ID of the fact to forget')`
- Calls `forgetFact(db, id)`
- Return JSON with `{ summary: 'Forgot fact N' | 'Fact N not found', data: { deleted: boolean } }`
- No auto-sync needed (write operation)

**kb_status** (status.ts):
- Input: none (empty schema `{}`)
- Queries DB for counts: repos, modules, events, services, edges, learned_facts
- Queries for stale repos: `SELECT name, path, last_indexed_commit FROM repos` then check each with `getCurrentCommit()` — but limit check to first 20 repos to avoid long startup (report "checked N of M")
- Return JSON with summary of counts + staleness info

**kb_cleanup** (cleanup.ts):
- Input: `prune: z.boolean().optional().describe('Actually delete data for missing repos (default: dry run)')`, `max_fact_age_days: z.number().optional().describe('Flag learned facts older than N days (default: 90)')`
- Call `detectDeletedRepos(db)` to find missing repos
- If `prune` is true, call `pruneDeletedRepos(db, deletedRepos)`
- Call `flagStaleFacts(db, max_fact_age_days)` to find old facts
- Return summary: "Found N deleted repos (pruned/not pruned), M stale facts flagged for review"
