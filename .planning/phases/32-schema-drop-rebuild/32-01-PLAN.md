**Step 2: Rewrite `src/db/schema.ts`**

Bump `SCHEMA_VERSION` to 10 (so existing v9 DBs trigger the rebuild path on first open).

Create a new exported function `createSchema(db)` that runs a single `db.exec()` creating ALL tables and indexes from scratch (no IF NOT EXISTS needed since the DB is clean):

Tables to create (consolidate from all 9 migrations into one DDL block):
- `repos` (id, name UNIQUE, path, description, last_indexed_commit, default_branch, created_at, updated_at)
- `files` (id, repo_id FK CASCADE, path, language, summary, created_at, updated_at, UNIQUE(repo_id, path))
- `modules` (id, repo_id FK CASCADE, file_id FK SET NULL, name, type, summary, table_name, schema_fields, created_at, updated_at)
- `services` (id, repo_id FK CASCADE, name, description, service_type, created_at, updated_at, UNIQUE(repo_id, name))

- `learned_facts` (id, content, repo, created_at)
- `fields` (id, repo_id FK CASCADE, parent_type, parent_name, field_name, field_type, nullable DEFAULT 1, source_file, module_id FK SET NULL, event_id FK SET NULL, created_at)

Indexes (from v1 + v5 + v8):
- `idx_edges_source` ON edges(source_type, source_id)
- `idx_edges_target` ON edges(target_type, target_id)
- `idx_edges_relationship` ON edges(relationship_type)
- `idx_modules_name` ON modules(name)
- `idx_events_name` ON events(name)
- `idx_services_name` ON services(name)

Rewrite `initializeSchema(db)`:
```
1. currentVersion = getCurrentVersion(db)
2. if currentVersion === SCHEMA_VERSION → call initializeFts(db) only, return (idempotent path)
3. if currentVersion === 0 → fresh DB: createSchema(db), setVersion(db, SCHEMA_VERSION), initializeFts(db), return
4. if currentVersion !== SCHEMA_VERSION → version mismatch, drop+rebuild:
   a. Export learned facts: SELECT id, content, repo, created_at FROM learned_facts (wrapped in try/catch — table may not exist in very old DBs)
   b. Get list of all user tables: SELECT name FROM sqlite_master WHERE type='table' AND name NOT LIKE 'sqlite_%'
   c. Disable foreign keys temporarily (PRAGMA foreign_keys = OFF)
   d. Drop all tables (DROP TABLE IF EXISTS for each, including FTS virtual tables like knowledge_fts and any _data/_idx/_content/_docsize/_config suffixed FTS shadow tables)
   e. Re-enable foreign keys (PRAGMA foreign_keys = ON)
   f. createSchema(db)
   g. setVersion(db, SCHEMA_VERSION)
   h. initializeFts(db)
   i. Re-import learned facts: INSERT INTO learned_facts (content, repo, created_at) VALUES (?, ?, ?) for each exported fact
      NOTE: IDs will change after rebuild — this is acceptable since facts are identified by content, not ID. The FTS re-indexing of facts happens on next `kb index` or `kb learn` call, not here. However, to be safe, also re-index each fact in FTS using indexEntity() from fts.ts.
```

IMPORTANT: The drop must handle FTS shadow tables. SQLite FTS5 creates shadow tables like `knowledge_fts_data`, `knowledge_fts_idx`, etc. Dropping `knowledge_fts` drops its shadow tables automatically, but we should drop the FTS virtual table explicitly before dropping regular tables to avoid FK issues.

IMPORTANT: Use `db.transaction()` to wrap the drop+rebuild+reimport for atomicity.
