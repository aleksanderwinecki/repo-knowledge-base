<task type="auto">
  <name>Task 2: Database module and complete schema with migrations</name>
  <files>src/db/database.ts, src/db/schema.ts, src/db/migrations.ts</files>
  <action>
Create the database module:

1. Create src/db/database.ts:
   - `openDatabase(dbPath: string): Database.Database` — creates parent directory with `fs.mkdirSync(dir, { recursive: true })`, opens better-sqlite3 Database, sets pragmas (journal_mode=WAL, foreign_keys=ON, synchronous=NORMAL), calls `initializeSchema(db)`, returns db.
   - `closeDatabase(db: Database.Database): void` — calls `db.close()`.
   - Register process exit handlers (SIGINT, SIGTERM) pattern in a `registerShutdownHandlers(db)` helper.
   - Export: `openDatabase`, `closeDatabase`.

2. Create src/db/schema.ts:
   - `const SCHEMA_VERSION = 1;`
   - `initializeSchema(db: Database.Database): void` — checks `db.pragma('user_version')`, if < SCHEMA_VERSION runs migrations in a transaction, sets user_version.
   - The CREATE TABLE statements (all with IF NOT EXISTS):
     - repos: id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT NOT NULL UNIQUE, path TEXT NOT NULL, description TEXT, last_indexed_commit TEXT, created_at TEXT NOT NULL DEFAULT (datetime('now')), updated_at TEXT NOT NULL DEFAULT (datetime('now'))
     - files: id, repo_id INTEGER NOT NULL REFERENCES repos(id) ON DELETE CASCADE, path TEXT NOT NULL, language TEXT, summary TEXT, created_at, updated_at, UNIQUE(repo_id, path)
     - modules: id, repo_id REFERENCES repos(id) ON DELETE CASCADE, file_id REFERENCES files(id) ON DELETE SET NULL, name TEXT NOT NULL, type TEXT, summary TEXT, created_at, updated_at
     - services: id, repo_id REFERENCES repos(id) ON DELETE CASCADE, name TEXT NOT NULL, description TEXT, created_at, updated_at, UNIQUE(repo_id, name)
     - events: id, repo_id REFERENCES repos(id) ON DELETE CASCADE, name TEXT NOT NULL, schema_definition TEXT, source_file TEXT, created_at, updated_at
     - edges: id, source_type TEXT NOT NULL, source_id INTEGER NOT NULL, target_type TEXT NOT NULL, target_id INTEGER NOT NULL, relationship_type TEXT NOT NULL, source_file TEXT, created_at TEXT NOT NULL DEFAULT (datetime('now'))
   - Indexes: idx_edges_source (source_type, source_id), idx_edges_target (target_type, target_id), idx_edges_relationship (relationship_type)
   - NOTE: FTS5 virtual table is NOT created here — it's in Plan 02.

3. Create src/db/migrations.ts:
   - `getCurrentVersion(db: Database.Database): number` — returns user_version pragma value
   - `setVersion(db: Database.Database, version: number): void` — sets user_version pragma
   - `runMigrations(db: Database.Database): void` — called by initializeSchema, runs version-gated migration blocks inside a transaction
   - For v1: all the CREATE TABLE/INDEX statements from schema.ts
  </action>
