**FIX-01: Implicit force for --repo**

In `src/indexer/pipeline.ts`, modify the staleness check guard (line 361) from:
```typescript
if (!options.force) {
```
to:
```typescript
if (!options.force && !options.repos?.length) {
```
This makes targeted repo reindex always bypass the staleness check. The `options.force` flag still works independently for full reindex.

In `src/cli/commands/docs.ts`, update the `kb index` section to document that `--repo` implies force:
- Add a line: `kb index --repo app-foo app-bar    # Reindex specific repos (always re-indexes)`
- Keep `--force` documented for the full-index use case.

**FIX-02: Symlink support in scanner**

In `src/indexer/scanner.ts`, change the directory check (line 29) from:
```typescript
if (!entry.isDirectory() || SKIP_DIRS.includes(entry.name)) {
```
to something like:
```typescript
if (SKIP_DIRS.includes(entry.name)) continue;
if (!entry.isDirectory() && !entry.isSymbolicLink()) continue;
```

Then resolve the actual path for symlinks before checking for .git and project files:
```typescript
const dirPath = path.join(rootDir, entry.name);

// For symlinks, verify target is a directory
if (entry.isSymbolicLink()) {
  try {
    const targetStat = fs.statSync(dirPath);
    if (!targetStat.isDirectory()) continue;
  } catch {
    continue; // Broken symlink
  }
}
```
