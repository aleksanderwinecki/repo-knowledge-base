2. Fix errors in each file using the patterns from RESEARCH.md. Apply these rules:

**Pattern A -- Regex match groups (elixir.ts, proto.ts, events.ts, graphql.ts, catalog.ts):**
For `match[1]`, `match[2]`, etc. inside while-loops where the regex already matched:
- Prefer guard-and-continue: `const name = match[1]; if (!name) continue;`
- Use `!` assertion only for structurally-guaranteed captures in non-optional groups where adding a guard would be awkward (e.g., `match[1]!` inside a `.map()` callback where continue is unavailable)

**Pattern B -- Parallel array indexing (pipeline.ts):**
For `workItems[i]` and `settled[i]` inside `for (let i = 0; i < settled.length; i++)`:
- Use `!` assertion: `const item = workItems[i]!;` and `const result = settled[i]!;`
- These are structurally guaranteed same-length parallel arrays

**Pattern C -- Array-by-index (elixir.ts):**
For `modulePositions[i]` and `modulePositions[i+1]` access:
- Use `!` assertion for `[i]` (loop bounds guarantee in-range)
- Use `!` assertion for `[i+1]` only when already guarded by length check

**Pattern D -- Record literal indexing (catalog.ts):**
For `idMatch[1]` inside ternary `idMatch ? idMatch[1] : s`:
- Use `idMatch[1]!` or `idMatch[1] ?? s`
