    - Test: getBranchCommit returns null for non-existent branch
    - Test: listBranchFiles returns all committed files from branch tree

    - Test: readBranchFile returns file content from the named branch
    - Test: readBranchFile returns content from main even when working tree has different content on feature branch

    - Test: getChangedFilesSinceBranch returns added/modified/deleted files between commit and branch tip
  </behavior>
  <action>
    Add 5 new exported functions to `src/indexer/git.ts`, following the established pattern (execSync with cwd, encoding, stdio pipe, try/catch returning null on failure):

    1. `resolveDefaultBranch(repoPath: string): string | null` -- tries `git rev-parse --verify refs/heads/main`, falls back to `git rev-parse --verify refs/heads/master`, returns null if neither exists. Hard-coded main/master only per user decision.

    2. `getBranchCommit(repoPath: string, branch: string): string | null` -- runs `git rev-parse refs/heads/${branch}`, returns trimmed SHA or null.

    3. `listBranchFiles(repoPath: string, branch: string): string[]` -- runs `git ls-tree -r --name-only ${branch}` with maxBuffer 10MB, splits output by newlines. Returns empty array on failure.

    4. `readBranchFile(repoPath: string, branch: string, filePath: string): string | null` -- runs `git show "${branch}:${filePath}"` with maxBuffer 500KB (matching existing MAX_FILE_SIZE). Quote the branch:path argument for safety against spaces. Returns null on failure.

    5. `getChangedFilesSinceBranch(repoPath: string, sinceCommit: string, branch: string): { added: string[]; modified: string[]; deleted: string[] }` -- runs `git diff --name-status ${sinceCommit}..${branch}`. Uses same parsing logic as existing `getChangedFiles` (handle A, M, D, R statuses). Returns empty lists on failure.

    Tests in `tests/indexer/git.test.ts`:
    - Use existing test pattern: create temp git repos with `mkdtempSync`, `git init`, `git config user.email/name`, write files, commit, branch rename.
    - Test helper: `createRepoWithBranch(name, branchName, files)` creates a temp repo on the specified branch.
    - Each function gets a `describe` block with the behavior tests listed above.
    - For readBranchFile branch isolation test: create repo on main, checkout feature branch, modify file, commit on feature -- then readBranchFile from "main" should return original content.
