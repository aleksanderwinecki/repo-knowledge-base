Output: Extended ElixirModule interface with new properties, pure parse functions returning structured data, comprehensive tests.

Existing parseElixirFile loop structure:
- Finds all `defmodule` positions via regex
- Slices `moduleContent` per module (from defmodule start to next defmodule or EOF)
- Calls extractModuledoc, extractPublicFunctions, extractSchemaTable on each slice
- Returns ElixirModule[] array

The extractSchemaTable function already exists (matches `schema "table_name"`) but only returns the table name string. Ecto field/association extraction extends this.

   - Object/input_object regex: `(object|input_object)\s+:(\w+)\s+do\b` -- captures kind and atom name.
   - Query/mutation root regex: `(query|mutation)\s+do\b` -- captures kind, name defaults to the kind itself ("query" or "mutation").

   - Deduplicate results. Return string[].

5. Integrate both into parseElixirFile loop. Call on each moduleContent slice.
