<feature>
  <name>Impact analysis module with tier classification, stats, and compact formatting</name>
  <files>src/search/impact.ts, tests/search/impact.test.ts</files>
  <behavior>
    analyzeImpact(db, serviceName, options?):
    - Builds graph, resolves serviceName to repoId, runs bfsUpstream
    - Returns ImpactResult with tiers.direct/indirect/transitive, stats, summary
    - Throws or returns error object when service not found
    - Passes mechanism filter and depth to bfsUpstream

    Tier classification:
    - depth 1 nodes -> direct array
    - depth 2 nodes -> indirect array
    - depth 3+ nodes -> transitive array (single bucket, no sub-groups)
    - Each entry: { name, mechanisms: string[], confidence: Array<string|null> }
    - mechanisms deduped (if node has 2 grpc edges, show "grpc" once)

    Stats computation:
    - total: direct.length + indirect.length + transitive.length
    - blastRadiusScore: direct.length * 3 + indirect.length * 2 + transitive.length * 1
    - mechanisms: Record<string, number> counting mechanism occurrences across ALL affected edges
    - Score does NOT factor in confidence (locked decision)
    - NO qualitative risk labels (locked decision)

    Summary string:
    - Format: "{service}: {N} direct, {N} indirect, {N} transitive (score: {N})"
    - Example: "app-orders: 5 direct, 3 indirect, 12 transitive (score: 33)"

    formatImpactVerbose(result):
    - Full JSON with all tiers, per-service confidence, all detail
    - Used by CLI (no size cap)

    formatImpactCompact(result):
    - Per-service: name -> mechanisms array (confidence dropped)
    - Always includes: summary, stats, all direct, all indirect
    - Transitive: fill until 4000-char budget exhausted, then "...and N more"
    - Budget for truncation message BEFORE calculating how many transitive services fit
    - Must handle edge case: 300+ affected services

    Error handling:
    - Service name not found in graph.repoIds -> throw Error("Service not found: {name}")

    Edge cases:
    - Service with zero upstream dependents -> empty tiers, score 0, summary "app-x: 0 direct, 0 indirect, 0 transitive (score: 0)"
    - Mechanism filter yields no results -> empty tiers
    - depth=1 -> only direct tier populated
  </behavior>
