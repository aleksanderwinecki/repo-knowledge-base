<feature>
  <name>bfsUpstream: reverse-adjacency BFS with mechanism filtering and multi-edge collection</name>
  <files>src/search/types.ts, src/search/graph.ts, src/search/index.ts, tests/search/graph.test.ts</files>
  <behavior>
    Core traversal:
    - bfsUpstream(graph, startRepoId) returns ImpactNode[] of all upstream dependents (services that call startRepoId, directly or transitively)
    - Starting node is NOT included in results (same pattern as bfsDownstream)
    - Each ImpactNode has depth: 1=direct caller, 2=caller's caller, etc.
    - Traverses graph.reverse (not graph.forward)
    - Does not traverse from unresolved nodes (repoId=0)
    - Handles cycles without infinite loops

    Mechanism filtering:
    - bfsUpstream(graph, startRepoId, 3, "grpc") only follows grpc edges during traversal
    - A -> B via grpc, B -> C via event: filtering for "grpc" returns only B (depth 1), NOT C
    - Without filter: returns both B and C

    Depth limiting:
    - Default maxDepth=3 (not Infinity like bfsDownstream -- per user decision)
    - bfsUpstream(graph, startRepoId, 1) returns only direct callers
    - bfsUpstream(graph, startRepoId, 2) returns direct and indirect callers

    Multi-edge collection:
    - Each ImpactNode.edges contains ALL edges from that node to any node in the BFS subgraph
    - If app-payments depends on app-orders via BOTH grpc and event, edges array has both
    - Edge collection uses graph.forward (affected node's outgoing edges that point into the subgraph)
    - When mechanism filter active, edge collection also respects the filter

    Edge cases:
    - Unknown repoId returns []
    - No upstream dependents returns []
    - Graph with only unresolved edges returns []
  </behavior>
  <implementation>
    1. Add ImpactNode interface to types.ts:
       ```typescript
       export interface ImpactNode {
         repoId: number;
         repoName: string;
         depth: number;
         edges: Array<{ mechanism: string; confidence: string | null }>;
       }
       ```

    2. Add bfsUpstream to graph.ts (after bfsDownstream):
       - Signature: `bfsUpstream(graph: ServiceGraph, startRepoId: number, maxDepth: number = 3, mechanismFilter?: string): ImpactNode[]`
       - BFS on graph.reverse with mechanism check before enqueueing
       - After BFS determines affected nodes, collect all edges from each affected node using graph.forward: scan forward edges that point to any node in the visited set (these are the dependency edges)
       - Apply mechanismFilter to edge collection too

    3. Update src/search/index.ts:
       - Add `bfsUpstream` to graph.ts re-exports
       - Add `ImpactNode` to type re-exports
  </implementation>
