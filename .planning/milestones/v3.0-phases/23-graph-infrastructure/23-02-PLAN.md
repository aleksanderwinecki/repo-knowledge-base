    Create src/search/graph.ts with three exported functions. Import extractConfidence, extractMetadataField from './edge-utils.js'. Import Database from 'better-sqlite3'. Import GraphEdge, ServiceGraph, BfsNode, GraphHop from './types.js'.

    **buildGraph(db: Database.Database): ServiceGraph**
    - Load repos: `SELECT id, name FROM repos` -> build repoNames (Map<number, string>) and repoIds (Map<string, number>)
    - Load direct edges: `SELECT source_id, target_id, relationship_type, metadata FROM edges WHERE source_type='repo' AND target_type='repo'` -> create GraphEdge for each, add to forward and reverse maps. Use extractConfidence(metadata) for confidence.
    - Load event producers: `SELECT DISTINCT e.source_id as repo_id, e.target_id as event_id, ev.name as event_name FROM edges e JOIN events ev ON ev.id = e.target_id WHERE e.relationship_type = 'produces_event'`
    - Load event consumers: `SELECT DISTINCT e.source_id as repo_id, e.target_id as event_id FROM edges e WHERE e.relationship_type = 'consumes_event'`
    - Resolve events in JS: Build Map<eventId, { repoId, eventName }[]> for producers, Map<eventId, number[]> for consumers. For each producer, for each consumer of same event, create edge with mechanism='event', via=eventName. Deduplicate by `${fromId}-${toId}-${eventName}`. Skip self-loops (fromId === toId).
    - Load kafka producers: `SELECT source_id as repo_id, json_extract(metadata, '$.topic') as topic, metadata FROM edges WHERE relationship_type = 'produces_kafka' AND json_extract(metadata, '$.topic') IS NOT NULL`
    - Load kafka consumers: `SELECT source_id as repo_id, json_extract(metadata, '$.topic') as topic, metadata FROM edges WHERE relationship_type = 'consumes_kafka' AND json_extract(metadata, '$.topic') IS NOT NULL`
    - Resolve kafka in JS: Build Map<topic, number[]> for producers and consumers. Same dedup/self-loop logic as events. mechanism='kafka', via=topicName.
    - Load unresolved: `SELECT source_id, relationship_type, metadata FROM edges WHERE source_type='repo' AND target_type='service_name'`. Add to forward map only with targetRepoId=0, extract targetName from metadata via extractMetadataField(metadata, 'targetName').
    - Return { forward, reverse, repoNames, repoIds }.

    **bfsDownstream(graph: ServiceGraph, startRepoId: number, maxDepth?: number): BfsNode[]**
    - Use graph.reverse adjacency list.
    - Queue: Array<[number, number]> (repoId, depth). Initialize with all reverse neighbors of startRepoId at depth 1.
    - visited: Set<number> initialized with startRepoId.
    - Process queue: skip repoId=0, skip visited, add to results as BfsNode (resolve name from graph.repoNames), enqueue reverse neighbors at depth+1 if depth < maxDepth (default Infinity).
    - Return collected BfsNode[].

    **shortestPath(graph: ServiceGraph, fromRepoId: number, toRepoId: number): GraphHop[] | null**
    - If fromRepoId === toRepoId, return [].
    - BFS with parent tracking: Map<number, { parentId: number, edge: GraphEdge, edgeIsForward: boolean }>.
    - Queue starts with fromRepoId. For each current node:
      - Forward neighbors: graph.forward.get(current) -> each edge's targetRepoId is neighbor, edgeIsForward=true
      - Reverse neighbors: graph.reverse.get(current) -> each edge's targetRepoId is neighbor (the repo that calls current), edgeIsForward=false
      - Skip visited, skip repoId=0
      - If neighbor === toRepoId, reconstruct path and return
    - Reconstruct: Walk parent pointers from toRepoId back to fromRepoId. For each step, if edgeIsForward: hop from=parentId, to=current. If not forward: hop from=current, to=parentId (showing actual call direction). Reverse the collected hops. Resolve repo names from graph.repoNames.
    - Return null if toRepoId never reached.

    Run tests -- all should pass. This is the GREEN phase.
